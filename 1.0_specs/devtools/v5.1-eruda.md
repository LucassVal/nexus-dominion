# SPEC: Eruda DevTools Integration — Nexus Dominion v5.1

## SDD (Spec-Driven Development)

---

## FEATURE 1: ERUDA CORE LOADER (Backend)

### Data
```
DEV_CONFIG = {
  enabled: bool,          // se Eruda esta ativo
  mode: 'offline'|'cdn',  // fonte do script
  plugins: ['monitor','timing','code','nd-inspector'],
  trigger: 'triple-tap'|'url'|'localStorage',
  theme: 'Dark'           // tema padrão
}
```

### Mechanics
1. Init flow:
   - Verificar `localStorage.getItem('nd_dev')` OU `?eruda=true` na URL
   - Se ativo: carregar Eruda core + plugins
   - Se nao: nao carregar nada (zero overhead em producao)
2. Offline mode (recomendado):
   - Baixar `eruda.min.js` (~100KB) + plugins (~15KB cada) pra pasta local
   - Referenciar como `<script src="eruda.min.js">`
   - Zero dados moveis apos download inicial
3. CDN fallback:
   - Se offline nao disponivel, carrega de `cdn.jsdelivr.net/npm/eruda@3`
4. Triple-tap trigger:
   - 3 toques rapidos no canto superior direito ativa/desativa Eruda
   - Alternativa: `?eruda=true` na URL (para bookmark)

### Integration
- Adicionado ANTES do `</body>` como ultimo script
- Nao interfere no jogo (Eruda vive em layer z-index:9999 separado)
- `eruda.init({tool: ['console','elements','network','resources','sources']})`

## FEATURE 2: GAME STATE INSPECTOR PLUGIN (Frontend)

### Data
```
ND_INSPECTOR = {
  sections: ['Player','World','Economy','Combat','AI'],
  refreshRate: 2000  // ms entre atualizacoes
}
```

### Mechanics
1. Custom Eruda plugin: `eruda.add({name:'ND-Inspector', ...})`
2. Painel mostra:
   - **Player tab**: HP, gold, lvl, xp, posicao, role, skills, equipment durability
   - **World tab**: dia atual, seed, clima, hora, cidades visitadas, pais
   - **Economy tab**: prices na cidade atual, supply/demand, inventario
   - **Combat tab**: ultimo combate (hp, dano, crit), abilities cooldown
   - **AI tab**: traders ativos, exercitos em guerra, bandits proximos
3. Atualizacao live: `setInterval(() => refreshPanel(), 2000)`
4. Botoes de cheat (só no dev mode):
   - `+10K Gold`, `+5 Levels`, `Heal`, `Teleport to city`

### UI
- Cada secao = bloco colapsavel
- Valores coloridos (gold=amarelo, hp=verde/vermelho)
- Progress bars para HP, XP, morale
- Tabela para inventario com quantidades

## FEATURE 3: PERFORMANCE MONITOR (UI)

### Data
```
PERF_MONITOR = {
  fps: 0,
  frameTime: 0,
  memMB: 0,
  entities: 0,
  particles: 0,
  lastGC: 0
}
```

### Mechanics
1. Instalar `eruda-monitor` plugin oficial (FPS + memoria)
2. Custom metricas via `eruda.get('monitor').addMetric(name, getter)`
3. Metricas customizadas:
   - `Entities`: cities.length + aiTraders.length + bandits.length + armies
   - `Particles`: particles.length
   - `PathLen`: G.player.path.length
   - `SaveKB`: JSON.stringify(saveData).length / 1024

### UI
- Overlay no topo-esquerdo (quando Eruda aberto)
- FPS: verde (>50), amarelo (30-50), vermelho (<30)
- Memoria: barra com % do limite
- Metricas customizadas em lista

## FEATURE 4: DEV MODE TOGGLE (UX)

### Data
```
DEV_TOGGLE = {
  active: bool,
  activationMethods: ['triple-tap','url-param','localStorage','bookmarklet'],
  hideInProd: true
}
```

### Mechanics
1. **Triple-tap**: 3 toques < 500ms no canto superior direito (50x50px) = toggle
2. **URL param**: `?eruda=true` ativa, `?eruda=false` desativa
3. **localStorage**: `localStorage.setItem('nd_dev','1')` = sempre ativo
4. **Bookmarklet**: `javascript:eruda.init()` em qualquer momento
5. Estado salvo em localStorage (`nd_dev_active`)
6. Em producao: Eruda completamente removido (build sem `<script>`)

### UI
- Botao sutil no top-right: icone de ferramenta (🔧) com opacidade 0.3
- Hover: opacidade 1.0
- Ao ativar: toast "DevTools ON" com fadeout 2s
- Ao desativar: toast "DevTools OFF"

## FEATURE 5: OFFLINE BUNDLE BUILDER (Backend)

### Data
```
OFFLINE_BUNDLE = {
  core: 'eruda.min.js',        // ~100KB
  plugins: {
    monitor: 'eruda-monitor.js',  // ~12KB
    timing: 'eruda-timing.js',    // ~10KB
    code: 'eruda-code.js',        // ~8KB
  },
  custom: 'nd-inspector.js',      // ~5KB (nosso plugin)
  totalSize: '~135KB'
}
```

### Mechanics
1. Script `build_devtools.py`:
   - Baixa Eruda core + plugins do CDN
   - Minifica com terser (ou ja vem minificado do CDN)
   - Concatena em `nexus-dominion-devtools.html` (versao com devtools)
   - Mantem `nexus-dominion.html` limpo (versao producao)
2. Make target: `make devtools-build` → gera versao com Eruda
3. Make target: `make devtools-clean` → remove scripts Eruda, volta pra prod

### Integration
- Script Python em `~/build/devtools/build_eruda_bundle.py`
- Output: `nexus-dominion-devtools.html` (com Eruda inline)
- Output: `nexus-dominion.html` (sem Eruda, production-ready)

## IMPLEMENTATION ORDER

1. **Fase A — Core offline**: Baixar eruda.min.js + plugins, salvar em `~/nexus-dominion/devtools/`
2. **Fase B — Loader condicional**: Script que detecta dev mode e carrega Eruda
3. **Fase C — ND-Inspector plugin**: Painel customizado de game state
4. **Fase D — Performance metrics**: eruda-monitor + metricas customizadas
5. **Fase E — Dev toggle UX**: Triple-tap, URL param, localStorage
6. **Fase F — Build system**: Script Python que gera versao dev vs prod
7. **Fase G — Documentacao**: README-dev.md com instrucoes de uso

Nao afeta save/load. Eruda eh camada de debug overlay, zero impacto no game state.
Todas as funcoes de cheat sao wrappers das funcoes existentes (G.player.gold += X, etc).
